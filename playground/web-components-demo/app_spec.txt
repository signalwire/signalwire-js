<product_requirements_document>

<overview>
SignalWire WebRTC Demo Application - A professional demonstration app showcasing the @signalwire/js SDK and @signalwire/web-components packages. The application provides a complete WebRTC communication experience including subscriber authentication, device management, directory-based calling, and dialpad functionality. Built with vanilla JavaScript and HTML templates, it serves as both a functional communication tool and a reference implementation for developers integrating SignalWire's WebRTC capabilities.
</overview>

<tech_stack>
   <frontend>
      - Vanilla JavaScript (ES modules) with HTML template tags for component rendering
      - Vite as development server and build tool
      - @signalwire/js SDK for WebRTC communication
      - @signalwire/web-components for pre-built Web Components
      - CSS custom properties for theming
      - No external UI frameworks
   </frontend>
   <backend>
      - None required - main only application
      - Token URL fetch performed directly by browser (assumes CORS-enabled endpoints)
   </backend>
</tech_stack>

<use_cases>

   <authentication>
      - App loads with welcome modal overlay blocking all other interactions
      - User sees two input options: direct SAT token input OR token URL input
      - A switch to toggle between "Enter Token" and "Fetch from URL" modes
      - In URL mode, user enters endpoint URL and app fetches JSON response expecting {"token": "..."}
      - Validation displays clear error messages for invalid tokens or failed URL fetches
      - On valid token, modal closes and main application interface becomes accessible
      - SignalWire initializes with provided token
   </authentication>

   <registration_management>
      - Top bar displays current registration status with visual indicator (green=registered, gray=unregistered)
      - Toggle switch allows user to register/unregister the subscriber
      - When registered, subscriber's display_name appears in top bar
      - Subscriber's available channels (audio/video/messaging) display as badges/icons
      - Status updates reactively via SDK observables
   </registration_management>

   <device_selection_with_preview>
      - Accessible via sidebar menu item "Select Devices"
      - Main content area displays sw-device-selector component
      - Three tabs: Microphone, Camera, Speaker
      - Camera tab shows live video preview
      - Microphone tab shows audio level indicator
      - Speaker tab provides test tone button
      - Device changes apply immediately to SDK
      - User can return to other views while preserving device selections
   </device_selection_with_preview>

   <directory_calling>
      - Accessible via sidebar menu item "Open Directory"
      - Sidebar expands to show secondary panel with sw-directory component
      - User can search addresses by name with debounced input
      - Address list shows name, type (person/room), and available channels
      - Clicking an address immediately initiates a call to that destination
      - Call UI takes over main content area upon dial
      - Directory panel remains accessible during call for reference
   </directory_calling>

   <dialpad_calling>
      - Accessible via sidebar menu item "Dialpad Calls"
      - Main content area displays sw-dialpad component with call button enabled
      - User enters destination via keypad or keyboard
      - Pressing call button or Enter initiates outbound call
      - During active call, dialpad allows DTMF digit sending
      - Call UI overlays/replaces dialpad view when call connects
   </dialpad_calling>

   <active_call_experience>
      - Call view occupies full available height of main content area
      - sw-call-media component displays remote video stream
      - sw-participants component overlays participant information on video
      - sw-self-media component shows local video in layout position (mirrored)
      - sw-call-controls component provides mute audio, mute video, screen share, hangup buttons
      - sw-call-status component shows call state and duration timer
      - Controls positioned at bottom of call view
      - Call status/duration visible at top of call view
      - Responsive layout adjusts for different viewport sizes
   </active_call_experience>

   <inbound_call_handling>
      - When subscriber receives inbound call, modal overlay appears
      - Modal displays caller information (name, number if available)
      - Two action buttons: Accept (green) and Reject (red)
      - If user is already on an active call, accepting new call automatically hangs up current call first
      - Rejecting dismisses modal and sends reject signal to caller
      - Accepting transitions to active call experience
      - Modal includes ringtone/visual pulse animation to attract attention
   </inbound_call_handling>

   <navigation>
      - Sidebar visible on left side of application
      - Collapsed state shows icons only with tooltips
      - Expanded state shows icons with labels
      - Toggle button at bottom of sidebar to collapse/expand
      - Three main menu items with distinct icons
      - Active menu item highlighted visually
      - Sidebar state does not persist between sessions
   </navigation>

</use_cases>

<model>

   <AppState>
      - currentView: 'devices' | 'directory' | 'dialpad' - tracks active main content
      - sidebarExpanded: boolean - controls sidebar collapse state
      - activeCall: Call | null - reference to current call if any
      - inboundCall: Call | null - pending inbound call awaiting user decision
      - isAuthenticated: boolean - whether valid token has been provided
   </AppState>

   <SubscriberInfo>
      - displayName: string - subscriber's display name from SDK
      - channels: {audio: boolean, video: boolean, messaging: boolean} - available channels
      - isRegistered: boolean - current registration status
      - isConnected: boolean - WebSocket connection status
   </SubscriberInfo>

   <CallState>
      - status: CallStatus - current call status from SDK observable
      - participants: Participant[] - list of call participants
      - self: SelfParticipant | null - local participant reference
      - localStream: MediaStream | null - local media stream
      - remoteStream: MediaStream | null - remote media stream
      - direction: 'inbound' | 'outbound' - call direction
      - callerInfo: {name: string, number: string} - for inbound call display
   </CallState>

   <AuthenticationState>
      - mode: 'token' | 'url' - selected input mode
      - tokenValue: string - direct token input
      - urlValue: string - URL to fetch token from
      - error: string | null - validation/fetch error message
      - isLoading: boolean - fetching token from URL
   </AuthenticationState>

</model>

<non_functional_requirements>
   - Application loads and becomes interactive within 3 seconds on broadband connection
   - All UI interactions provide immediate visual feedback (< 100ms)
   - Graceful handling of WebSocket disconnection with reconnection attempts
   - Clear error messages for all failure scenarios (network, permissions, invalid input)
   - Accessible: keyboard navigable, ARIA labels on interactive elements
   - Responsive: functional on viewports from 768px width and above (tablet/desktop focus)
   - Browser support: Latest versions of Chrome, Firefox, Safari, Edge
   - Media permissions requested only when needed (lazy permission request)
   - Console logging for debugging without exposing sensitive token data
   - Clean disconnection on page unload (beforeunload handler)
</non_functional_requirements>
